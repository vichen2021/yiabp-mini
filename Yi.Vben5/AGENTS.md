# Yi.Vben5 Agents 协作规范

本文档是 `Yi.Vben5` 管理端前端的 AI Agent 协作规范。处理本目录任务时，应同时遵循仓库根目录 `AGENTS.md`。

## ⚠️ 最高优先级警告：基础 CRUD 必须由 Skill 生成

- **用户不信任手写的基础 CRUD 代码**：前端 `api/` 与 `views/` 中的基础列表/表单/弹窗脚手架，必须由后端 `crud-generator-plus`（含前端联动产物）生成，未经 skill 的手写产物用户都视为不可信。
- **一旦发现绕过 skill 手写**：用户会**整体回退所有相关代码**（前端页面、API、类型、字典常量都会一并回退），无论看起来多"能跑"、多"干净"。
- **后果非常严重**：开发进度直接拖慢、已经消耗的大量 token 全部作废、前后端契约被迫重做、信任成本大幅上升。
- **不允许任何替代方案**：不要因为"skill 生成结果不完美"、"只是小改动"、"先快速跑通"就跳过 skill；需要修正请基于 skill 产物增量调整，不要另起炉灶。
- **谨记**：手写 CRUD = 高概率被整体回退 = 浪费 token = 浪费时间，动手前先确认是否应使用 skill。

## 工作区规则

- **优先打开仓库根目录**：默认以 `yiabp-mini` 为工作区，不要只打开 `Yi.Vben5` 子目录。
- **先识别影响范围**：开始修改前先判断是否只涉及管理端前端，或需要同步后端 DTO、枚举、字典、权限、接口路径。
- **最小范围变更**：只修改与当前需求直接相关的文件，避免无关格式化、重构、依赖升级和配置变更。
- **先查现有实现**：新增页面、组件、API、表格、表单前，先搜索同类模块实现，沿用现有模式。
- **保持前后端契约一致**：后端 DTO、查询条件、枚举、字典、权限、动态 API 路径变化后，必须同步前端类型和调用代码。

## 技术栈约束

- **框架**：Vue 3 + TypeScript。
- **管理模板**：Vben Admin。
- **UI 组件**：Ant Design Vue。
- **表格**：VXE Table，优先使用项目封装的 `useVbenVxeGrid`。
- **表单**：优先使用项目封装的 `useVbenForm` 和 schema 写法。
- **请求**：统一使用 `#/api/request` 中的 `requestClient`。
- **状态和工具**：优先使用项目已有 composable、utils、adapter，不重复造轮子。

## API 规则

- **路径遵循后端动态 API**：不要自行添加模块名前缀或发明接口路径。
- **API 文件位置**：按模块放在 `apps/web-antd/src/api/{module}/`。
- **类型文件位置**：接口类型放在同目录 `model.d.ts`。
- **分页返回**：分页接口应使用项目已有 `PageResult`、`PageQuery` 等公共类型。
- **批量删除**：删除接口参数遵循项目现有 `IDS` 写法。
- **下拉接口**：优先沿用后端生成器约定的下拉数据接口路径。

### 常见接口路径约定

一般情况下，前端 API 以实体短横线名称作为 `Api.root`，例如 `user` 对应 `/user`，`file-record` 对应 `/file-record`。常见接口路径如下：

| 场景 | 方法 | 路径 |
|------|------|------|
| 分页列表 | `GET` | `${Api.root}` |
| 普通列表 | `GET` | `${Api.root}/list` |
| 详情 | `GET` | `${Api.root}/{id}` |
| 新增 | `POST` | `${Api.root}` |
| 更新 | `PUT` | `${Api.root}/{id}` |
| 批量删除 | `DELETE` | `${Api.root}?ids={ids}` |
| 树表列表 | `GET` | `${Api.root}/tree` |
| 下拉框 | `GET` | `${Api.root}/select-data-list` |
| 导入 | `POST` | `${Api.root}/import` |
| 导出 | `GET` | `${Api.root}/export` |
| 下载导入模板 | `GET` | `${Api.root}/import-template` |

若后端接口由生成器或动态 API 约定产生，应以后端实际路径为准，不要为了前端习惯自行改名。

## 页面开发规则

- **先参考同类页面**：新增 CRUD 页面前，先参考已有模块的 `index.vue`、`data.ts`、`*-drawer.vue` 写法。
- **页面职责清晰**：`index.vue` 负责表格、查询、工具栏、弹窗/抽屉挂载；`data.ts` 负责查询 schema、表格列、表单 schema；抽屉或弹窗负责新增编辑交互。
- **组件复用优先**：跨页面复用的选择器、上传器、业务弹窗应放到模块内 `components/` 或合适的共享组件目录。
- **避免内联复杂逻辑**：复杂事件处理函数应使用具名函数，并补齐 TypeScript 类型。
- **权限指令同步**：按钮权限使用 `v-access:code`，权限码应与后端权限定义和菜单种子保持一致。

## 类型和字典规则

- **保持类型同步**：后端 DTO 字段变化后，同步 API 类型、列表列、搜索项、表单项。
- **避免隐式 any**：事件参数、表格行、回调参数应尽量显式标注类型。
- **枚举优先走字典**：后端枚举进入系统字典后，前端使用 `DictEnum`、`getDictOptions`、`renderDict` 等现有工具。
- **布尔状态统一**：普通启停状态优先使用项目已有状态字典和渲染方式。

## 表单和弹窗规则

- **新增编辑优先使用抽屉**：CRUD 新增/编辑场景优先沿用现有 drawer 模式。
- **短流程操作可使用弹窗**：上传、选择、批量操作等短流程可使用 modal。
- **提交前校验**：调用新增/更新 API 前必须执行表单校验。
- **树形父级空值统一处理**：后端用空 Guid 表示顶级/无父级时，前端编辑展示前必须使用 `emptyGuidToNull(parentId)` 转为空值，避免 `TreeSelect` 回显裸 `00000000-0000-0000-0000-000000000000`；提交前必须使用 `toGuidOrEmpty(parentId)` 转回空 Guid。不要为了匹配空 Guid 在下拉树中额外添加“顶级”假节点，除非业务数据本身真实存在该节点。
- **提交后刷新**：新增、更新、删除、批量操作成功后，应刷新当前表格。
- **关闭前差异检查**：编辑抽屉优先使用项目已有关闭前差异检查工具。

## 依赖和配置规则

- **禁止无关依赖变更**：不要随意修改 `package.json`、lockfile、构建配置、lint 配置。
- **禁止绕过封装**：不要在业务代码中绕过已有请求、权限、表格、表单封装，除非已有封装无法满足需求且已明确说明原因。
- **禁止大范围格式化**：不要对无关文件做格式化或重排导入。

## 验证要求

- **前端变更**：至少检查类型、路由、页面引用和必要的 type-check/lint。
- **API 变更**：检查接口路径、请求方法、参数命名、返回类型与后端一致。
- **权限变更**：检查菜单、按钮权限码、权限定义和前端 `v-access:code` 是否一致。
- **提交前检查**：使用 `git status --short` 确认没有混入无关文件。

## 禁止事项

- **禁止未确认需求就大范围重构**。
- **禁止自行发明后端不存在的接口路径**。
- **禁止修改与任务无关的包版本、锁文件、构建脚本和格式化配置**。
- **禁止把一次性业务逻辑放入全局公共工具目录**。
- **禁止提交未验证的类型错误和明显的隐式 any 问题**。
