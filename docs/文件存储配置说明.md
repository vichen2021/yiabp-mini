# 文件存储配置说明

## 默认配置

项目默认使用**本地文件系统**存储文件，配置路径为 `wwwroot/FileStorage`。

开发者克隆项目后无需额外配置即可本地运行，文件会保存到本地目录。

## 切换到阿里云 OSS

生产环境可切换到阿里云 OSS 存储。在 `appsettings.json` 或环境变量中配置：

```json
{
  "BlobStoring": {
    "Provider": "Aliyun",
    "Aliyun": {
      "AccessKeyId": "your-access-key-id",
      "AccessKeySecret": "your-access-key-secret",
      "Endpoint": "oss-cn-hangzhou.aliyuncs.com",
      "ContainerName": "your-bucket-name",
      "CreateContainerIfNotExists": false
    }
  }
}
```

### 配置项说明

| 配置项 | 说明 |
|--------|------|
| `Provider` | 存储提供者，可选值：`FileSystem`（默认）、`Aliyun` |
| `Aliyun:AccessKeyId` | 阿里云 AccessKeyId |
| `Aliyun:AccessKeySecret` | 阿里云 AccessKeySecret |
| `Aliyun:Endpoint` | OSS Endpoint，如 `oss-cn-hangzhou.aliyuncs.com` |
| `Aliyun:ContainerName` | OSS Bucket 名称 |
| `Aliyun:CreateContainerIfNotExists` | Bucket 不存在时是否自动创建，建议 `false` |

## 安全注意事项

### 密钥管理

**禁止将真实密钥提交到 Git 仓库！**

推荐方式：

- **本地开发**：使用 `appsettings.Development.json` 或用户机密配置
- **Docker / 生产环境**：使用环境变量覆盖配置
- **CI/CD**：使用 Secret 注入

### RAM 权限

建议为项目单独创建 RAM 用户，授权范围限制到指定 Bucket：

- 读取 Object
- 写入 Object
- 删除 Object
- 查询 Bucket 基础信息

**不建议使用主账号 AccessKey。**

### Bucket 访问权限

建议 Bucket 设置为**私有读写**：

- 当前系统通过后端接口下载文件
- 权限校验由系统控制
- 避免上传文件被公开访问

## 接口兼容性

切换存储 Provider 后，文件上传、下载、删除接口保持不变：

- 上传：`POST /api/file/upload`
- 下载：`GET /api/file/get/{id}`
- 删除：`DELETE /api/file/{id}`

前端无需适配，后端继续代理文件访问，保证权限控制和接口兼容。

## Provider 选择建议

| 场景 | 推荐 Provider |
|------|---------------|
| 本地开发 | `FileSystem` |
| 生产部署（小规模） | `FileSystem` 或 `Aliyun` |
| 生产部署（大规模） | `Aliyun`（配合 CDN） |
| 多服务器部署 | `Aliyun`（避免本地文件同步问题） |

## 注意事项

- Provider 配置忽略大小写，`aliyun` 和 `Aliyun` 均可
- 若配置非法或无法识别，系统回退到 `FileSystem`
- 切换 Provider 后，旧文件可能无法读取（需重新上传）
- 本项目不做双写，不做历史数据迁移工具
