# 阿里云短信基础发送模块设计方案

## 1. 背景

当前项目已有阿里云短信发送能力，但代码位于 `Rbac` 业务模块中：

```text
Yi.Abp/module/rbac/Yi.Module.Rbac.Domain/Managers/AliyunSmsManager.cs
```

短信发送本质上是基础设施能力，不属于 `Rbac`、`Member` 或其他具体业务模块。

如果其他业务模块为了发送短信而引用 `Rbac.Domain`，会导致业务模块之间产生不合理依赖。

因此，需要将阿里云短信发送能力从业务模块中抽离，沉淀为框架层基础能力。

---

## 2. 设计目标

本次只设计基础短信发送模块。

目标：

- 提供可复用的阿里云短信发送能力。
- 从 `Rbac` 业务模块中解耦短信基础设施代码。
- 避免业务模块之间为了短信发送互相引用。
- 统一短信配置模型。
- 保持阿里云短信配置与阿里云 OSS 配置分离。
- 支持配置完整性校验。
- 支持阿里云短信业务返回码校验。
- 日志不输出敏感密钥。

---

## 3. 非目标

本次不涉及：

- 不实现 `UniApp` 登录。
- 不实现会员短信验证码登录。
- 不实现账号注册、找回密码、绑定手机号等业务流程。
- 不设计短信验证码生成。
- 不设计短信验证码缓存。
- 不设计短信验证码校验。
- 不设计短信发送记录表。
- 不设计短信频控风控。
- 不实现多短信供应商切换。
- 不实现腾讯云、华为云短信。

本模块只负责：

```text
给定手机号和模板参数，调用阿里云短信服务完成发送。
```

---

## 4. 模块位置

建议新增框架集成模块：

```text
Yi.Abp/framework/integrations/Yi.Framework.Aliyun.Sms/
```

项目名：

```text
Yi.Framework.Aliyun.Sms
```

模块类：

```csharp
YiFrameworkAliyunSmsModule
```

依赖方向：

```text
Yi.Framework.Aliyun.Sms
  <- Yi.Module.Rbac
  <- 其他业务模块
```

业务模块只能依赖短信基础模块，不能通过依赖其他业务模块间接获得短信能力。

---

## 5. 职责边界

### 5.1 基础短信模块职责

`Yi.Framework.Aliyun.Sms` 负责：

- 绑定 `AliyunOptions` 配置。
- 注册短信发送服务。
- 创建阿里云短信 SDK Client。
- 发送短信。
- 校验必要配置。
- 校验阿里云返回码。
- 记录发送失败日志。

### 5.2 基础短信模块不负责

`Yi.Framework.Aliyun.Sms` 不负责：

- 生成验证码。
- 缓存验证码。
- 校验验证码。
- 判断手机号是否存在。
- 控制发送频率。
- 处理具体业务流程。

这些逻辑应留在业务模块中。

---

## 6. 接口设计

### 6.1 当前阶段接口

当前阶段先保持阿里云语义，降低迁移成本：

```csharp
public interface IAliyunSmsManager
{
    Task SendSmsAsync(string phoneNumber, string code);
}
```

实现类：

```csharp
public class AliyunSmsManager : IAliyunSmsManager
```

### 6.2 接口语义

`SendSmsAsync` 的职责：

- 使用默认短信签名。
- 使用默认验证码模板。
- 传入手机号。
- 传入验证码。
- 发送模板参数：

```json
{
  "code": "123456"
}
```

该接口适合当前已有验证码短信场景。

### 6.3 后续扩展方向

如果后续需要支持不同模板、不同短信用途，可扩展为：

```csharp
public interface IAliyunSmsManager
{
    Task SendSmsAsync(string phoneNumber, string templateCode, object templateParam);
}
```

当前阶段不引入，避免扩大改造范围。

---

## 7. 配置设计

### 7.1 配置节点

继续使用顶层配置：

```json
"AliyunOptions": {
  "AccessKeyId": "",
  "AccessKeySecret": "",
  "Sms": {
    "SignName": "",
    "TemplateCode": ""
  }
}
```

### 7.2 Options 类型

建议放入短信基础模块：

```csharp
public class AliyunOptions
{
    public string AccessKeyId { get; set; } = string.Empty;
    public string AccessKeySecret { get; set; } = string.Empty;
    public AliyunSmsOptions Sms { get; set; } = new();
}

public class AliyunSmsOptions
{
    public string SignName { get; set; } = string.Empty;
    public string TemplateCode { get; set; } = string.Empty;
}
```

### 7.3 与 OSS 配置分离

短信配置：

```text
AliyunOptions
```

OSS 配置：

```text
BlobStoring:Aliyun
```

两者不合并。

原因：

- OSS 当前属于 BlobStoring Provider 配置。
- 短信属于独立基础集成能力。
- 两者生命周期、使用场景、配置校验不同。

---

## 8. 模块注册设计

`YiFrameworkAliyunSmsModule` 负责配置绑定：

```csharp
Configure<AliyunOptions>(configuration.GetSection(nameof(AliyunOptions)));
```

`AliyunSmsManager` 可通过 ABP 自动依赖注册：

```csharp
public class AliyunSmsManager : IAliyunSmsManager, ITransientDependency
```

或继承合适的 ABP 基类并显式实现依赖注册接口。

---

## 9. 发送实现设计

### 9.1 创建 Client

使用：

```text
AlibabaCloud.SDK.Dysmsapi20170525
```

Endpoint：

```text
dysmsapi.aliyuncs.com
```

### 9.2 发送参数

发送请求包含：

- `PhoneNumbers`
- `SignName`
- `TemplateCode`
- `TemplateParam`

验证码模板参数：

```json
{
  "code": "123456"
}
```

### 9.3 配置校验

发送前校验：

- `AccessKeyId`
- `AccessKeySecret`
- `Sms.SignName`
- `Sms.TemplateCode`

缺失时抛出明确异常：

```text
阿里云短信配置不完整，请检查 AliyunOptions 配置节点
```

### 9.4 返回码校验

阿里云 SDK 调用成功不代表短信业务成功。

需要校验：

```text
response.Body.Code == "OK"
```

非 `OK` 时记录：

- `Code`
- `Message`
- `RequestId`

不记录：

- `AccessKeyId`
- `AccessKeySecret`

---

## 10. 迁移设计

### 10.1 从 Rbac 迁移

当前 `Rbac.Domain` 中的：

```text
AliyunSmsManager
IAliyunSmsManager
AliyunOptions
```

应迁移到：

```text
Yi.Framework.Aliyun.Sms
```

迁移后：

- `Rbac.Domain` 移除阿里云短信 SDK 依赖。
- `Rbac.Domain` 引用 `Yi.Framework.Aliyun.Sms`。
- 现有业务调用接口名称保持 `IAliyunSmsManager`。

### 10.2 配置位置

`appsettings.json` 中保留空配置示例。

真实密钥放入：

```text
appsettings.Development.json
```

或部署环境变量、私有配置中心。

---

## 11. 实施计划

### 阶段一：新增框架短信模块

1. 创建 `Yi.Framework.Aliyun.Sms` 项目。
2. 创建 `YiFrameworkAliyunSmsModule`。
3. 引入阿里云短信 SDK。
4. 添加 `AliyunOptions`。
5. 添加 `IAliyunSmsManager`。
6. 添加 `AliyunSmsManager`。

验收标准：

- 框架短信模块单独编译通过。
- 模块可绑定 `AliyunOptions`。
- DI 可解析 `IAliyunSmsManager`。

### 阶段二：迁移 Rbac 现有调用

1. `Rbac.Domain` 引用 `Yi.Framework.Aliyun.Sms`。
2. 移除 `Rbac.Domain` 中原有短信 Manager。
3. 移除 `Rbac.Domain` 中阿里云短信 SDK 包引用。
4. 确认 `AccountService` 仍可注入 `IAliyunSmsManager`。

验收标准：

- `Rbac` 编译通过。
- 现有短信发送调用不变。
- `Rbac` 不再直接承载阿里云短信基础设施实现。

### 阶段三：构建验证

执行：

```text
dotnet build Yi.Abp.slnx --no-restore
```

验收标准：

- 解决方案编译通过。
- 无业务模块反向依赖。
- 无真实密钥进入仓库。

---

## 12. 风险与注意事项

### 12.1 模板变量

当前默认发送参数为：

```json
{ "code": "123456" }
```

阿里云短信模板必须包含：

```text
${code}
```

### 12.2 敏感信息

不得提交真实：

- `AccessKeyId`
- `AccessKeySecret`

### 12.3 模块边界

任何业务模块不得为了短信发送依赖另一个业务模块。

正确方式：

```text
业务模块 -> Yi.Framework.Aliyun.Sms
```

错误方式：

```text
业务模块 -> Rbac.Domain -> AliyunSmsManager
```

---

## 13. 结论

阿里云短信发送应作为基础设施能力沉淀到：

```text
Yi.Framework.Aliyun.Sms
```

当前阶段只封装基础发送能力。

业务模块只负责自己的业务流程，例如验证码生成、缓存、校验、账号或会员处理。

基础短信模块只负责把短信发送出去，并对配置和阿里云返回结果进行可靠校验。
