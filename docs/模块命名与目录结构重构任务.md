# 模块命名与目录结构重构任务

## 背景

当前 `Yi.Abp/module` 下的业务模块项目使用 `Yi.Framework.*` 前缀，例如：

- `Yi.Framework.Rbac.*`
- `Yi.Framework.AuditLogging.*`
- `Yi.Framework.TenantManagement.*`
- `Yi.Framework.SettingManagement.*`

这些项目属于可插拔业务/功能模块，不属于底层框架基础设施。继续使用 `Yi.Framework.*` 容易和 `Yi.Abp/framework` 下的框架能力混淆。

本次重构目标是明确边界：

- `Yi.Framework.*`：只表示框架基础设施、通用能力、横切能力。
- `Yi.Module.*`：表示业务模块、功能模块、可插拔模块。

## 目标

1. 将 `Yi.Abp/module` 下模块项目从 `Yi.Framework.*` 前缀迁移为 `Yi.Module.*`。
2. 将 `Yi.Abp/framework` 下同前缀项目按功能目录归组。
3. 保持 API 行为、权限码、操作日志行为不变。
4. 将重命名和目录归组拆成两个独立提交，便于回滚和排查。

## 非目标

本次不调整以下内容：

- 不修改业务 API 路由。
- 不修改权限码规则。
- 不修改操作日志规则。
- 不修改数据库结构。
- 不调整模块依赖边界。
- 不重命名 `Yi.Abp/src` 下应用主项目。
- 不将 `Yi.Framework.Operation.*` 改为模块，它仍属于框架横切能力。

## 当前工作区注意事项

当前工作区存在以下已知变动：

- `Yi.Abp/src/Yi.Abp.Web/yi-abp-dev.db`

该文件属于运行期数据库文件，不纳入本次重构任务。执行迁移时不得覆盖、还原或提交该文件，除非后续明确要求。

## 第一阶段：模块前缀迁移

### 迁移范围

仅迁移 `Yi.Abp/module` 下有 `.csproj` 的模块项目：

- `audit-logging`
- `rbac`
- `setting-management`
- `tenant-management`

`module/sec-log` 当前未发现 `.csproj`，暂不纳入迁移。若确认是废弃目录，应单独清理。

### 命名规则

执行以下精确映射：

```text
Yi.Framework.Rbac              -> Yi.Module.Rbac
Yi.Framework.AuditLogging      -> Yi.Module.AuditLogging
Yi.Framework.TenantManagement  -> Yi.Module.TenantManagement
Yi.Framework.SettingManagement -> Yi.Module.SettingManagement
```

同步修改：

- 文件夹名
- `.csproj` 文件名
- namespace
- using
- ABP Module 类名
- ProjectReference
- `.slnx` 项目路径
- 测试项目引用
- Client/Web/Application/Domain/SqlSugarCore 聚合模块引用

### ABP Module 类名规则

将 `YiFramework{Module}{Layer}Module` 改为 `YiModule{Module}{Layer}Module`。

示例：

```text
YiFrameworkRbacApplicationModule
-> YiModuleRbacApplicationModule

YiFrameworkAuditLoggingDomainModule
-> YiModuleAuditLoggingDomainModule
```

## 第二阶段：framework 目录归组

### 原则

只调整目录结构，不修改以下内容：

- 项目名
- `.csproj` 文件名
- namespace
- ABP Module 类名
- 程序集名

这样可以降低影响面，同时让 `framework` 目录更清晰。

## 风险与处理

### ProjectReference 路径遗漏

风险：目录或 `.csproj` 文件重命名后，引用路径未同步。

处理：构建后按错误逐项修复，并使用 `rg "Yi.Framework.Rbac|Yi.Framework.AuditLogging|Yi.Framework.TenantManagement|Yi.Framework.SettingManagement"` 检查残留。

### ABP Module 类名遗漏

风险：`DependsOn(typeof(...))` 引用旧 Module 类名。

处理：统一替换 `YiFramework{Module}` 为 `YiModule{Module}`，并构建验证。

### 路由或权限码变化

风险：命名空间变化影响自动推断逻辑。

处理：当前权限解析主要依据 service 类型、命名空间片段和方法名。迁移后需回归 CRUD 接口、权限校验和操作日志。若发现模块名推断变化，应在 `DefaultActionMetadataResolver` 中补齐 `Yi.Module.*` 命名空间识别。

### Windows 文件锁

风险：Visual Studio、Web 进程或测试进程锁定 `bin/obj`。

处理：关闭 IDE 和运行进程。必要时使用独立输出目录构建验证。
